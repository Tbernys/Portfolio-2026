**1. buildTileBackground()** — Creates the cylindrical/hemispherical tile arrangement.

Create a single `InstancedMesh` using `CylinderGeometry` (short, wide: radiusTop ≈ radiusBottom ~0.45, height ~0.08, radialSegments: 8) for the tile look.

Tile arrangement — hemisphere/sphere distribution at large radius (~8 units) around the camera using Fibonacci sphere algorithm for even distribution. Approximately 120 tiles (medium density per user decision). Each tile faces inward (`dummy.lookAt(0, 0, 0)`) so the flat face is visible from inside.

**ShaderMaterial with GLSL3** (per research Pitfall 7 — must use `glslVersion: THREE.GLSL3` to access `gl_InstanceID`):

Uniforms:
- `uTime` (float) — elapsed time for animation
- `uTileCount` (float) — total tile count for hash normalization

Vertex shader:
- Pass `float(gl_InstanceID)` as varying `vId` to fragment shader
- Pass world position as varying for potential use
- Standard MVP transform with `instanceMatrix`

Fragment shader — per-tile pattern selection based on hash of `vId`:
- Use a simple hash function: `fract(sin(vId * 12.9898) * 43758.5453)` to classify each tile
- ~40% noise pattern: Animated simplex/value noise using vId as seed offset + uTime for flow. Inline GLSL noise (~15-20 lines of value noise or simplex noise, NO external library per research).
- ~30% gradient pattern: Smooth gradient across tile face, angle/direction varies per tile, slowly rotates over time
- ~30% solid pattern: Single muted color per tile, with subtle brightness pulsing over time

Color palette — muted, low-saturation per user decision (Claude's discretion on exact values):
- Navy: `vec3(0.08, 0.10, 0.18)`
- Charcoal: `vec3(0.12, 0.12, 0.14)`
- Deep teal: `vec3(0.06, 0.14, 0.16)`
- Slate: `vec3(0.10, 0.11, 0.15)`
- Mix between these using hash of vId for variety

Animation — "flowing motion, visible movement at comfortable pace, patterns ripple and evolve noticeably" per user decision:
- Noise patterns: offset by `uTime * 0.3` (not too fast, not static)
- Gradient patterns: rotate angle by `uTime * 0.15`
- Solid patterns: pulse brightness with `sin(uTime * 0.5 + vId) * 0.05`

Set `side: THREE.DoubleSide` on the material so tiles are visible from both sides if needed.

Add tiles to scene. Store `tiles` and `tileMat` at module level for RAF updates.

**2. buildCrosshairs()** — Creates crosshair marks at tile grid intersections (Claude's discretion on styling per CONTEXT.md).

After computing tile positions in buildTileBackground, identify logical grid intersection points. For the Fibonacci sphere arrangement, crosshairs can be placed at the same positions as tiles but rendered as thin cross shapes.

Use a second `InstancedMesh` or `Points` geometry:
- Small cross shapes (~0.05 unit size) at each tile center position
- Subtle color (slightly brighter than surrounding tiles, e.g., `vec3(0.2, 0.22, 0.25)`)
- Thin line aesthetic — use `LineSegments` with `LineBasicMaterial` for each cross, or a simple cross-shaped `PlaneGeometry` with a cross texture/shader
- Alternatively: use the same positions as tiles but offset slightly, with a flat ShaderMaterial that draws a cross pattern in the fragment shader (cleanest approach for instancing)

Add crosshairs to scene.

- Do NOT set `instanceMatrix.needsUpdate = true` every frame (positions are static per research anti-pattern)

**1. initInteraction()** — Sets up pointer/touch event listeners.

Module-level state:
```javascript
const pointer = { x: 0, y: 0 };
const targetRot = { x: 0, y: 0 };
let isIdle = true;
let idleTimer = null;
```

Event handler `onPointerMove(e)`:
- Normalize clientX/clientY to [-1, 1] range (handle both mouse and touch via `e.touches ? e.touches[0].clientX : e.clientX`)
- Compute target rotation: `targetRot.y = pointer.x * maxAngle` and `targetRot.x = pointer.y * maxAngle * 0.5` where `maxAngle = Math.PI / 6` (~30° per user decision "medium rotation range ~25-30°")
- Set `isIdle = false`, clear and restart idle timer
- Idle timer fires after ~150ms of no movement → sets `isIdle = true`

Register listeners:
- `window.addEventListener('pointermove', onPointerMove)` — covers mouse
- `window.addEventListener('touchmove', e => onPointerMove(e), { passive: true })` — covers touch. Use `passive: true` to avoid scroll jank.

**2. updateInteraction(delta)** — Called every RAF frame. Lerps textMesh rotation toward target.

When `isIdle = false` (active interaction):
- `lerpSpeed = 0.08` — fast, responsive follow ("smooth follow, magnetically attracted" per user decision)

When `isIdle = true` (no interaction):
- Drift targetRot toward {0, 0}: `targetRot.x *= 0.99; targetRot.y *= 0.99;`
- `lerpSpeed = 0.015` — slow drift back ("lazy, dreamy return to center" per user decision, ~3-4s)

Apply lerp:
- `textMesh.rotation.y += (targetRot.y - textMesh.rotation.y) * lerpSpeed`
- `textMesh.rotation.x += (targetRot.x - textMesh.rotation.x) * lerpSpeed`

**3. updateFloat(clock)** — Called every RAF frame. Vertical bob animation.

- `textMesh.position.y = baseY + Math.sin(clock.getElapsedTime() * 0.6) * 0.08`
- Where `baseY` is the text's original Y position (likely 0)
- Amplitude 0.08 and frequency 0.6 give a calm, serene float per user decision ("always alive but serene")
- Claude's discretion on exact values — tune for "calm vertical bob"

**4. Wire into existing code:**
- Call `initInteraction()` in `main()` after scene is built
- Add `updateInteraction()` and `updateFloat(clock)` calls in the RAF loop, AFTER `updateTiles(clock)`
- Guard all updates: `if (textMesh) { updateInteraction(); updateFloat(clock); }`

**5. Context loss recovery:**
- Event listeners survive context loss (they're on window, not canvas)
- `textMesh` reference is refreshed in `initScene()` rebuild, so interaction continues working after restore
