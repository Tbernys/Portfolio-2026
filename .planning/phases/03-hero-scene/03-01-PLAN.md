
**1. setupEnvMap(renderer, scene)** — Call BEFORE building text mesh.
- Create PMREMGenerator from renderer
- Generate env texture from `new RoomEnvironment()`
- Set `scene.environment = envTexture`
- Dispose PMREMGenerator after generating texture

**2. loadFont()** — async function.
- Use `new FontLoader().loadAsync('fonts/comfortaa_bold.typeface.json')` (Option A from research — fetch from same origin, cached)
- Return the Font object
- Note: The context decision says "serif font" for classic/editorial feel. However, the project already has Comfortaa Bold (rounded sans-serif) pre-converted at `fonts/comfortaa_bold.typeface.json` — this is the ONLY font available in typeface.json format. The 850-glyph conversion was done in Phase 1 specifically for Phase 3 TextGeometry. Use Comfortaa Bold per the existing project asset. The serif preference is overridden by the practical constraint — the font is already converted and ready.

**3. buildTextMesh(font)** — Creates the 3D glass text.
- Create `TextGeometry('Tom Bernys', { font, size, depth: 0.4, curveSegments: 6, bevelEnabled: true, bevelThickness: 0.03, bevelSize: 0.02, bevelSegments: 4 })`
- Call `geometry.computeBoundingBox(); geometry.center();` — critical for correct rotation pivot
- Scale text to approximately 60% of viewport width. Compute analytically:
  - Visible width at z=0 for PerspectiveCamera: `visibleWidth = 2 * cameraZ * Math.tan(fov/2 * Math.PI/180)`
  - Measure text bounding box width after centering
  - Set `textMesh.scale.setScalar(targetWidth / bboxWidth)` where `targetWidth = visibleWidth * 0.6`
- Create `MeshPhysicalMaterial({ transmission: 1.0, ior: 1.5, thickness: 0.5, roughness: 0.0, metalness: 0.0, transparent: true, envMapIntensity: 1.0 })` — crystal clear glass, pure neutral color, no tint per user decision
- Create mesh and add to scene
- Store reference in module-level `textMesh` variable for interaction (Plan 03-02)

**4. Lighting setup** (Claude's discretion per CONTEXT.md).
- Add 2-3 point lights or directional lights to create edge highlights on the glass
- Use subtle warm/cool tones to give depth without tinting the glass
- Position lights to catch the sharp bevel edges per user decision ("light catching distinct angles")
- Consider an ambient light at low intensity for baseline fill

**5. setupPostProcessing(renderer, scene, camera)** — Creates EffectComposer pipeline.
- `new EffectComposer(renderer)`
- Add `RenderPass(scene, camera)` as first pass
- Add `UnrealBloomPass(new THREE.Vector2(w, h), strength, radius, threshold)` — tune for "subtle haze bloom" per user decision. Start with strength ~0.4, radius ~0.3, threshold ~0.75. These are tunable.
- Add `ShaderPass(VignetteShader)` — use the built-in VignetteShader from Three.js addons, NOT a hand-rolled shader. Set `darkness` uniform to ~1.5 for "medium vignette, clearly visible darkening" per user decision.
- Add `OutputPass()` as FINAL pass — critical in r175, without it colors are wrong
- Store `composer` and `bloomPass` at module level

**6. Wire into existing code:**
- In the existing `handleResize()` function, add `if (composer) composer.setSize(w, h);` after `renderer.setSize()`
- In the RAF loop, replace `renderer.render(scene, camera)` with `if (composer) composer.render(); else renderer.render(scene, camera);` — the fallback ensures rendering works during init before composer is ready
- In the `main()` bootstrap sequence:
  - Call `setupEnvMap(renderer, scene)` after `initScene()`
  - Load font: `const fontReady = loadFont()` — this runs in parallel with other warmup
  - After font loads, call `buildTextMesh(font)` and add to scene
  - All hero geometry MUST be added to scene BEFORE `warmupShaders()` is called (per Pitfall 5 from research) — this ensures compileAsync covers the glass material
  - Update the loading gate to include font loading: add fontReady to the Promise.all array alongside shadersReady

**7. Context loss recovery:**
- In the existing `initScene()` (which is re-callable on context restore), ensure the env map and text mesh are rebuilt. The simplest approach: call setupEnvMap + buildTextMesh inside initScene if font is already loaded (store font reference at module level). Rebuild composer in setupPostProcessing after renderer recreation.
