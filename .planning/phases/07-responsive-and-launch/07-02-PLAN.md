  <action>
This task requires the user to provide a GitHub repo URL since no remote is currently configured (`git remote get-url origin` returns empty).

**Steps for the user:**

1. Create a GitHub repository (if not already done) — public or private, name it something like `portfolio-2026`
2. Share the repo URL with Claude (e.g., `https://github.com/tombernys/portfolio-2026.git`)

**Once the URL is provided, Claude will execute:**

```bash
# Add GitHub remote
git remote add origin <USER_PROVIDED_URL>

# Push main branch
git push -u origin main

# Create gh-pages branch with only needed files (per D-18)
git checkout --orphan gh-pages
git reset --hard
git checkout main -- index.html
git checkout main -- og-image.jpg 2>/dev/null  # OG image if exists
git checkout main -- fonts/ 2>/dev/null         # Fonts directory if exists
touch .nojekyll
git add -A
git commit -m "chore: initial GitHub Pages deployment"
git push -u origin gh-pages
git checkout main
```

The `.nojekyll` file prevents Jekyll processing (standard practice per research). The `gh-pages` branch contains only the files needed to serve the site. This is the chosen approach per locked decision D-18 (deploy via gh-pages branch).

After pushing, the user must enable GitHub Pages in repo Settings: **Settings > Pages > Source: Deploy from branch > gh-pages > / (root)**.

> **Note:** An alternative would be serving from `main` branch directly (Settings > Pages > Source: `main`), which avoids the orphan branch but exposes all repo files. This is NOT the chosen approach — D-18 specifies gh-pages branch.
