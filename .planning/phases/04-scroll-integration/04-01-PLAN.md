  <name>Task 1: Fix touch events and wire ScrollTrigger canvas fade with RAF optimization</name>
  <files>index.html</files>
  <action>
Three changes in index.html:

**1. Import ScrollTrigger and register it.**
Near the existing `import { gsap } from 'gsap';` (line ~279), add:
```javascript
import { ScrollTrigger } from 'gsap/ScrollTrigger';
gsap.registerPlugin(ScrollTrigger);
```

**2. Fix touch event handlers (lines ~1906-1921).**
The current `window`-level touchstart/touchmove listeners call `e.preventDefault()` which blocks ALL native scrolling on mobile. Replace them with canvas-scoped passive listeners:

- Remove the three `window.addEventListener('touchstart/touchmove/touchend', ...)` blocks (lines ~1906-1921).
- Add replacement listeners on the canvas element:
  - `canvasEl.addEventListener('touchstart', ...)` with `{ passive: true }` — NO preventDefault. Just return early if target is A/BUTTON.
  - `canvasEl.addEventListener('touchmove', ...)` with `{ passive: true }` — read `touch.clientX/clientY` into `mouseX/mouseY` for glass text interaction. NO preventDefault. This means touch simultaneously scrolls AND rotates the glass text, which is the desired behavior.
  - `canvasEl.addEventListener('touchend', ...)` — add `hoverVelY += 0.03` velocity impulse as before.
- Use `const canvasEl = document.getElementById('canvas');` (or reuse the existing canvas reference if one exists in scope).

**3. Create the ScrollTrigger canvas fade tween.**
After the touch event setup (or in a new `setupScrollIntegration()` function called from `main()`), add:

```javascript
let heroScrollProgress = 0;

gsap.to('#canvas', {
  opacity: 0,
  scale: 0.95,
  scrollTrigger: {
    trigger: '#hero',
    start: 'top top',
    end: 'bottom top',
    scrub: true,
    onUpdate: (self) => {
      heroScrollProgress = self.progress;
      if (self.progress > 0.95 && rafId !== null) {
        stopRAFLoop();
      } else if (self.progress <= 0.95 && rafId === null && !document.hidden) {
        startRAFLoop();
      }
    }
  }
});
```

Make sure `heroScrollProgress`, `rafId`, `stopRAFLoop`, and `startRAFLoop` are accessible in this scope (they are module-level variables already).

Also ensure the `#canvas` element has `will-change: transform, opacity` in its CSS rule for GPU-accelerated compositing during the scroll tween.

**Placement:** Call `setupScrollIntegration()` from `main()` AFTER `startRAFLoop()` — ScrollTrigger needs the page to be laid out before it measures trigger positions.

**3. Add ScrollTrigger section triggers** in the `setupScrollIntegration()` function (after the canvas fade tween):

```javascript
const sections = ['hero', 'clients', 'projects', 'contact'];
sections.forEach((id) => {
  ScrollTrigger.create({
    trigger: `#${id}`,
    start: 'top center',
    end: 'bottom center',
    onToggle: (self) => {
      if (self.isActive) {
        document.querySelectorAll('.nav-dot').forEach(d => d.classList.remove('active'));
        const dot = document.querySelector(`.nav-dot[data-section="${id}"]`);
        if (dot) dot.classList.add('active');
      }
    }
  });
});
```
