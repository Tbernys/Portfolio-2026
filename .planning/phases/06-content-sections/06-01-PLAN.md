**A) Add `initContactForm()` function** in the JS `<script type="module">` block, placed right after the `initProjectsShowcase()` function definition (after the closing brace of initProjectsShowcase). The function:

```javascript
function initContactForm() {
  const form = document.getElementById('contact-form');
  if (!form) return;

  form.addEventListener('submit', async (e) => {
    e.preventDefault();
    const feedback = document.getElementById('form-feedback');
    const submitBtn = form.querySelector('.contact-submit');

    // Clear previous feedback (Pitfall 4)
    feedback.textContent = '';
    feedback.className = 'form-feedback';

    // Disable button during submission
    submitBtn.disabled = true;
    submitBtn.textContent = 'Envoi...';

    try {
      const response = await fetch(form.action, {
        method: 'POST',
        body: new FormData(form),
        headers: { 'Accept': 'application/json' }
      });

      if (response.ok) {
        feedback.textContent = 'Message envoy\u00e9 ! Je vous r\u00e9ponds rapidement.';
        feedback.className = 'form-feedback form-feedback--success';
        form.reset();
      } else {
        const json = await response.json().catch(() => ({}));
        const msg = json.errors?.map(err => err.message).join(', ') || 'Une erreur est survenue.';
        feedback.textContent = msg;
        feedback.className = 'form-feedback form-feedback--error';
      }
    } catch {
      feedback.textContent = 'Erreur r\u00e9seau. Veuillez r\u00e9essayer.';
      feedback.className = 'form-feedback form-feedback--error';
    } finally {
      submitBtn.disabled = false;
      submitBtn.textContent = 'Envoyer';
    }
  });
}
```

Key implementation details:
- `headers: { 'Accept': 'application/json' }` is MANDATORY — without it Formspree returns a 302 redirect (Pitfall 1)
- Feedback cleared at start of each submission (Pitfall 4)
- Button disabled during fetch to prevent double-submit
- `form.action` used as fetch URL (single source of truth, not hardcoded)
- `form.reset()` called on success to clear fields
- Per D-09: fetch (no redirect). Per D-10: success text. Per D-11: error text. Per D-12: no custom JS validation (HTML5 only).
