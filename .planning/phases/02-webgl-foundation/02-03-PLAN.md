<task type="auto">
  <name>Task 1: Implement shader warmup gate and glitch dissolve transition</name>
  <files>index.html</files>
  <action>
In the `<script type="module">` block, add the transition system and wire it into the bootstrap.

1. **warmupShaders()** — async function:
   - Call `renderer.compileAsync(scene, camera)` and return the Promise.
   - This pre-compiles all shaders in the current scene during the loading screen.
   - Currently the scene has minimal objects (Plan 01 placeholder), but this establishes the pattern. Phase 3 will add hero geometry BEFORE calling this, so the compile covers all hero materials.

2. **transition()** function — the glitch dissolve:
   - **Step A: Kill GSAP tweens safely** (prevents Pitfall 5):
     - `gsap.killTweensOf('.loader-char')` and kill the stored loader timeline.
   - **Step B: Snap progress bar to 100%:**
     - Set `.loader-progress-bar` width to `100%`.
   - **Step C: Glitch dissolve effect:**
     - Create a GSAP timeline that fragments the `#loader` overlay using `clip-path` animation.
     - Approach: Split the loader into 4-6 rectangular fragment zones using CSS `clip-path: polygon(...)`. Animate each fragment translating/rotating away in staggered sequence over ~1s.
     - Alternative simpler approach (Claude's discretion): Use a rapid sequence of CSS `clip-path: inset(...)` keyframes that progressively reveal more of the canvas underneath, combined with opacity fade and a slight scale transform on the loader text — creating a "tearing apart" visual.
     - The effect should feel like digital fragmentation, matching the corruption/glitch aesthetic of the text reveal — NOT a clean opacity fade.
     - Duration: ~1s total (per user decision: "medium — readable but doesn't linger").
   - **Step D: Set `pointer-events: none` on #loader immediately** when transition starts (so the hero is interactive during the dissolve animation).
   - **Step E: Remove #loader from DOM** after the dissolve animation completes (use GSAP `onComplete` callback or `transitionend` event). This prevents the overlay pseudo-elements from consuming compositing resources.

3. **Promise-based readiness gate** — wire into `main()`:
   ```javascript
   const minDisplay = new Promise(r => setTimeout(r, 2000));
   const shadersReady = warmupShaders();
   const fallback = new Promise(r => setTimeout(r, 5000));

   Promise.race([
     Promise.all([minDisplay, shadersReady]),
     fallback
   ]).then(() => transition());
   ```
   - This ensures: (A) loading screen shows for at least 2s, (B) shaders compile before hero reveal, (C) 5s maximum wait.

4. **Bootstrap update:**
   - The `main()` function should now:
     1. `gpuTier = await detectTier()`
     2. `initRenderer()`
     3. `initScene()`
     4. `startRAFLoop()` — renderer runs behind overlay
     5. `initLoadingScreen(gpuTier)` — starts text scramble + effects
     6. Set up the Promise gate (minDisplay + shadersReady + fallback → transition)
   - The RAF loop runs during loading so compileAsync has a valid GL context.

5. **CSS addition for the dissolve:**
   - Add any necessary CSS for the clip-path fragment animation.
   - Ensure `#loader.dissolving` (or equivalent state class) triggers the fragment effect.
