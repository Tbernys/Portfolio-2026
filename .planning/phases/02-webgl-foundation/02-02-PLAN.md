**CSS for #loader** — add to the existing `<style>` block:

1. `#loader`: position fixed, inset 0, z-index 100 (above everything), background #000, display flex, align-items center, justify-content center, flex-direction column. `transition: opacity 0.3s ease-out` (for later fade).

2. `.loader-text`: display flex, gap 0. Font: JetBrains Mono 700 (monospace fits the decryption aesthetic per research Open Question 2 recommendation). `font-size: clamp(2rem, 6vw, 4.5rem)`. Color white. `text-transform: uppercase`.

3. `.loader-char`: display inline-block, `min-width: 0.7em` (fixed slot width prevents layout shift during scramble — research recommendation). `text-align: center`.

4. `.loader-space`: `min-width: 0.4em` (narrower for space character).

5. **Scanline sweep** (`#loader::after`):
   - `content: ''`, position absolute, inset 0, pointer-events none.
   - Background: `linear-gradient(to bottom, transparent 45%, rgba(255,255,255,0.03) 49%, rgba(255,255,255,0.08) 50%, rgba(255,255,255,0.03) 51%, transparent 55%)`.
   - `animation: scanline 2s linear infinite`.
   - Initially hidden — add class `.has-scanline` to enable.

6. **Film grain** (`#loader::before`):
   - `content: ''`, position absolute, inset 0, pointer-events none, opacity 0.04.
   - Background: an inline SVG data URI using `<feTurbulence type="fractalNoise" baseFrequency="0.9" numOctaves="4" />` as a filter applied to a rect.
   - `animation: grain 0.3s steps(4) infinite`.
   - Initially hidden — add class `.has-grain` to enable.

7. **@keyframes scanline**: `from { transform: translateY(-100%); } to { transform: translateY(200%); }`.

8. **@keyframes grain**: 4-step animation shifting `transform: translate()` by small random-looking offsets (e.g., 0%: translate(0,0), 25%: translate(-2%,4%), 50%: translate(4%,-2%), 75%: translate(-4%,1%), 100%: translate(0,0)).

9. **Progress bar**:
   - `.loader-progress`: position absolute, bottom 2rem (or top 2rem — Claude's discretion), left 50%, transform translateX(-50%), width 200px, height 2px, background rgba(255,255,255,0.1), border-radius 1px.
   - `.loader-progress-bar`: height 100%, width 0%, background var(--color-accent), border-radius 1px, `transition: width 0.3s ease-out`.

The scanline and grain classes (`.has-scanline`, `.has-grain`) will be toggled by JS based on gpuTier in Task 2.

<task type="auto">
  <name>Task 2: Implement GSAP ScrambleText character reveal and tier-aware loading activation</name>
  <files>index.html</files>
  <action>
In the `<script type="module">` block, add the loading screen logic AFTER detectTier/initRenderer/initScene but BEFORE startRAFLoop.

1. **Import GSAP and ScrambleTextPlugin:**
   ```javascript
   import { gsap } from 'gsap';
   import { ScrambleTextPlugin } from 'gsap/ScrambleTextPlugin';
   gsap.registerPlugin(ScrambleTextPlugin);
   ```

2. **initLoadingScreen(tier)** function:
   - Get all `.loader-char` elements.
   - If `tier >= 2`: add classes `.has-scanline` and `.has-grain` to `#loader`.
   - Start the GSAP ScrambleText timeline.

3. **GSAP ScrambleText timeline** — `runGlitchReveal()`:
   - Create `gsap.timeline()`.
   - For each `.loader-char` element (index `i`):
     - `tl.to(el, { duration: 0.6, scrambleText: { text: el.dataset.char, chars: '!@#$%^&*<>{}[]|\\/?~_+=', revealDelay: 0.3, speed: 0.4 } }, i * 0.2)`.
   - The stagger of 0.2s per character means: 10 chars * 0.2s + 0.6s = ~2.6s total (within the ~2.5s target).
   - Skip the space character (`.loader-space`) — just set its text to a space directly without scramble.
   - Return the timeline (for later `kill()` on transition).

4. **Progress bar animation:**
   - Create a separate GSAP tween that animates `.loader-progress-bar` width from `0%` to `90%` over 4.5s with `ease: "power1.out"`. The last 10% jump to 100% happens when the transition triggers (Plan 03).
   - This gives a smooth progress feel without needing real progress data.

5. **Update the bootstrap `main()` function:**
   - After `detectTier()`, `initRenderer()`, `initScene()`:
   - Call `initLoadingScreen(gpuTier)` — this starts the scramble animation.
   - Do NOT call `startRAFLoop()` yet — Plan 03 will start it after the transition.
   - Actually, DO start the RAF loop (renderer needs to be running for `compileAsync` in Plan 03 to work), but the loader overlay visually covers it.

6. **Export/expose the GSAP timeline** so Plan 03's transition code can `tl.kill()` it before removing the overlay (preventing the GSAP error from Pitfall 5 in research).
   - Store as `window._loaderTimeline = tl;` or a module-level variable.
