<task type="auto">
  <name>Task 1: Add GSAP and detect-gpu to import map, implement tier detection and production renderer</name>
  <files>index.html</files>
  <action>
Update the import map in index.html to add these entries:
- `"gsap": "https://cdn.jsdelivr.net/npm/gsap@3.14.2/index.js"`
- `"gsap/": "https://cdn.jsdelivr.net/npm/gsap@3.14.2/"`
- `"detect-gpu": "https://cdn.jsdelivr.net/npm/detect-gpu@5.0.70/dist/detect-gpu.esm.js"`

Replace the entire `<script type="module">` block. Remove the Phase 1 test cube code entirely. Build the new module structure:

1. **Imports:** Import THREE, getGPUTier from detect-gpu.

2. **detectTier()** — async function:
   - Call `await getGPUTier()` wrapped in try/catch (fallback to tier 1 on error).
   - Return the tier number (0-3).
   - Store tier in a module-level variable (`let gpuTier = 1`).

3. **initRenderer()** — function:
   - Get the existing `#canvas` element.
   - Create `WebGLRenderer({ canvas, antialias: gpuTier >= 2, alpha: false })`. Disable antialias on low-tier devices.
   - `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))`.
   - `renderer.setSize(window.innerWidth, window.innerHeight)`.
   - Store renderer in module-level variable.

4. **initScene()** — function:
   - Create a new `THREE.Scene()`.
   - Create `PerspectiveCamera(40, aspect, 0.1, 100)` at z=5.
   - Add a simple ambient light (0xffffff, 0.5) — placeholder for Phase 3 hero scene lighting.
   - Store scene and camera in module-level variables.
   - This function is designed to be re-callable on context restore.

5. **RAF loop** with visibility guard:
   - `let rafId = null;`
   - `startRAFLoop()`: if rafId exists return; set up `requestAnimationFrame` loop calling `renderer.render(scene, camera)`.
   - `stopRAFLoop()`: `cancelAnimationFrame(rafId); rafId = null;`
   - `document.addEventListener('visibilitychange', ...)`: if hidden → stopRAFLoop(), else → startRAFLoop().

6. **Context loss/restore handlers:**
   - `canvas.addEventListener('webglcontextlost', () => { stopRAFLoop(); })` — do NOT call preventDefault (Three.js already does this internally in r175).
   - `canvas.addEventListener('webglcontextrestored', () => { initScene(); startRAFLoop(); })` — re-create scene objects after Three.js rebuilds GL context.

7. **Resize handler:**
   - `window.addEventListener('resize', ...)`: update `camera.aspect`, call `camera.updateProjectionMatrix()`, call `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))` then `renderer.setSize(w, h)`.
   - Do NOT create new WebGLRenderTarget instances in the resize handler. Phase 2 has no render targets yet — this handler just covers renderer + camera. The pattern is established so Phase 3 can add `renderTarget.setSize(w, h)` calls.

8. **Bootstrap:**
   - `async function main() { gpuTier = await detectTier(); initRenderer(); initScene(); startRAFLoop(); console.log('[Phase 2] GPU tier:', gpuTier, '| Three.js r' + THREE.REVISION); }`
   - Call `main()`.

Keep the canvas rendering a blank/black scene for now — no geometry. The loading screen overlay (Plan 02) will cover it during init.
